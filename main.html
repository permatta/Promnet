GIT TOPIK

GIT adalah sebuah perangkat lunak yang digunakan untuk mengelola versi kode program dengan cara menentukan baris serta kode yang perlu diganti atau ditambahkan. GIT adalah perangkat lunak yang digunakan oleh Linus Torvald untuk membuat kernel Linux pada tahun 2005. Selanjutnya, GIT dikelola oleh Jumino Hamano, seorang programmer asal Jepang, dan saat ini digunakan untuk mengelola source code. Git adalah sistem kontrol (Version Control System) yang sangat populer di kalangan pengembang perangkat lunak karena kecepatan, keamanan, dan kemampuan untuk mendukung kerja kolaboratif. Ini adalah alat yang sangat penting untuk mengelola kode sumber proyek perangkat lunak.

Kelebihan GIT

  • Mempermudah untuk mengetahui perubahan pada source codenya.
  • Memfasilitasi kolaborasi real-time dengan rekan pengembang untuk melakukan pembaruan kode.
  • Memungkinkan partisipasi yang mudah dari para pengembang dalam proses pengembangan yang sedang berlangsung.
  • Git dapat digunakan untuk mendokumentasikan desain.
  • Git memiliki antarmuka pengguna yang sederhana dan mudah digunakan.

Perintah Dasar GIT

  • git init: untuk membuat repository pada file lokal.
  • git status: untuk mengetahui status dari repository lokal.
  • git add: untuk menambahkan file ke staging area.
  • git commit: untuk menyimpan perubahan pada repository lokal.
  • git push: untuk mengirimkan perubahan pada repository lokal ke repository online.
  • git pull: untuk mengambil perubahan dari repository online ke repository lokal.
  • git clone: untuk membuat salinan repository lokal.
  • git checkout: untuk menukar branch yang aktif dengan branch yang sudah dipilih.
  • git reset: untuk mengembalikan keadaan workdir ke keadaan commit yang diinginkan tanpa menambah commit baru.
  • git branch: perintah untuk membuat, menampilkan, dan menghapus branch dalam repository git.
  • git fetch: untuk mengambil commit terbaru dari remote repository.
